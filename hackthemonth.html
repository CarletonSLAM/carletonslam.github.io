November Hack The Month Challenge

Build An API

Judjing Day: Decmber 1st

November's challenge is to create an API to be consumed by someone else. One way to build an API is to use a Rails/Django/Node.js application which can be hosted using a service such as Heroku to allow others to consume your API. In your web app you will create different routes that will become your different API calls. Using a web app framework you can also store data to your API using a database for things like user information or metrics if applicable. Another possible use of an API would be to return JSON containing the information needed for building a web page dynamically using javascript or any front end framework of your choice. For this challenge Documenting the different API calls will be important as well as documenting expected return values so that the APIs can be used by other people. This documentation should go in the README on the github project.

There is no front end required for this project we are simply looking at the API that you build. Judging will be based on how well the API works and how well it is documented. Having multiple commands is helpfull but not nescesary. Error handling is also helpful but unnescesary.

October Hack The Month Challenge

Consume An API

Judjing Day: November 3rd

October's challange was to create a program that would consume an API. There are different kinds of APIs like REST APIs which use static web calls to return or alter data using GET and PUT requests respectively, and Graph APIs which are accessed in a similar way to that of REST but you can define specific fields that you would like to return. Some examples of different APIs available are the Twitter(REST) and Facebook(Graph) APIs. The goal of this challange is to consume the data from an API (Not nescesarilly these two) and do something interesting/creative/functional with it.
